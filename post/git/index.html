Git

2020-03-22 3 min read # Git

定义

是一个分布式版本控制工具（系统）。

Git特点

1.尽可能添加数据而不是删除或修改数据，存储能力强，容易备份

2.完整性保证（对于每次提交做Hash的操作）

3.与linux命令全面兼容

4.良好性能，支持离线（在本地完成）

5.制定工作流

6.分布式仓库

Git官网

https://git-scm.com/book/zh/v2

Git三种状态,三个逻辑区域

三种状态

  • modified(已修改)

  • staged(已暂存)

  • committed(已提交)

三种分区

git分为：工作区，暂存区，本地库，远程区 四块大区域

对应关系

.Git文件内容结构

Git命令

搭建Git仓库

两种场景：

使用init初始化，将本地的目录变成git本地仓库

  • 1.已有项目，将项目纳入Git管理
cd 项目所在文件夹
git init
  • 2.没有项目，新建用Git管理的项目
cd 项目所在文件夹
git init your_project   #会在当前路径下创建和项目名称同名的文件夹

查看本地目录的状态

git status

本地仓库与远程仓库关联

与远程仓库关联，（此前已在码云等上注册远程仓库）

git add origin 远程仓库地址

更新项目

准备上传你的项目之前，最好更新一下项目，确保没有和远程仓库的代码有冲突

git pull --rebase origin master

提交项目

git add .           #添加目录下所有发生改变的文件
git commit -m           #'注释信息'
git push -u origin master       #本地仓库代码提交至远程仓库(有时候会提交失败，可以尝试git push -u origin master -f 强制提交,只建议在第一次提交中使用)

克隆项目

git clone 远程仓库地址

分支相关命令

  • 查看所有分支
git branch  -a
  • 创建分支
git branch 分支名
  • 切换到某一分支
git checkout  分支名
  • 合并分支
git merge 分支名
  • 创建远程的分支,并且把本地分支推送到远程分支上
git push origin 分支名
  • 删除本地分支
git branch -d 分支名
  • 查看本地分支状态
git branch
  • 删除远程分支
git push origin --delete [branch_name]
